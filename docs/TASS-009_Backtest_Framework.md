# TASS-009 : Backtest Framework

## (백테스트 프레임워크)

**Version:** v0.1.0

**Status:** Draft

**Author:** TASS Project

---

# 1. 목적 (Purpose)

본 문서는 TASS의 모든 Rule, Score, Gate, Confidence, Risk가 반드시 거쳐야 하는 백테스트 기준을 정의한다.

TASS는 직관이나 경험이 아닌 데이터로 검증된 규칙만 사용한다.

---

# 2. 핵심 철학

백테스트는 높은 수익률을 만드는 것이 목적이 아니다.

목적은

* Rule 검증
* Score 검증
* Weight 검증
* Risk 검증

이다.

---

# 3. 검증 원칙

모든 Rule은

설계

↓

구현

↓

단위 테스트

↓

백테스트

↓

채택

순서를 반드시 따른다.

백테스트를 수행하지 않은 Rule은 운영에 사용할 수 없다.

---

# 4. 테스트 대상

다음을 모두 검증한다.

* Rule
* Score
* Weight
* Gate
* Confidence
* Risk
* Recommendation Engine

---

# 5. 테스트 데이터

시장

* KOSPI
* KOSDAQ

기간

최소 10년

권장

15년 이상

봉 기준

일봉

---

# 6. 데이터 품질

백테스트 데이터는 다음 조건을 만족해야 한다.

* 결측치 없음
* 수정주가 반영
* 액면분할 반영
* 거래정지 처리
* 상장폐지 처리
* 날짜 정렬 완료

---

# 7. 거래 조건

기본 가정

* 종가 기준 진입
* 다음 거래일 시초 진입(선택)
* 수수료 반영
* 세금 반영
* 슬리피지 반영

현실적인 거래 비용을 포함한다.

---

# 8. 성과 지표

반드시 측정한다.

* 총 수익률
* 연평균 수익률(CAGR)
* 승률
* 평균 수익
* 평균 손실
* 손익비(R:R)
* Profit Factor
* 최대 낙폭(MDD)
* Sharpe Ratio
* Sortino Ratio
* 거래 횟수

---

# 9. Rule 평가

Rule 하나를 추가하면

기존 시스템과 비교한다.

검증 항목

* 승률 변화
* 수익률 변화
* MDD 변화
* 거래 횟수 변화

개선되지 않으면 채택하지 않는다.

---

# 10. Weight 평가

Weight 변경 시

반드시 전체 백테스트를 다시 수행한다.

감으로 Weight를 변경하지 않는다.

---

# 11. Gate 평가

Gate 추가 시

확인한다.

* 제거 종목 비율
* 제거 후 성과
* 제거 후 승률
* 제거 후 MDD

효과가 없으면 제거한다.

---

# 12. Confidence 평가

Confidence별 성과를 비교한다.

예시

95~100%

90~94%

80~89%

70~79%

각 구간별

* 승률
* 평균 수익률
* 손익비

를 기록한다.

---

# 13. Risk 평가

Risk Score별 성과를 비교한다.

낮은 Risk가 실제로

더 좋은 성과를 내는지 검증한다.

---

# 14. 비교 대상

새로운 Rule은 반드시

기존 버전과 비교한다.

예시

v0.1

↓

v0.2

↓

성과 비교

---

# 15. 테스트 로그

모든 백테스트는 기록한다.

기록 항목

* 날짜
* 버전
* Rule
* 데이터
* 결과
* 변경 내용

---

# 16. 재현성

동일한 데이터

동일한 설정

동일한 버전

이라면

항상 동일한 결과가 나와야 한다.

---

# 17. 실패 기준

다음 중 하나라도 발생하면 Rule을 재검토한다.

* 승률 감소
* MDD 증가
* 손익비 악화
* 과최적화 의심
* 거래 횟수 과도 감소

---

# 18. 과최적화 방지

특정 기간에만 잘 작동하는 Rule은 채택하지 않는다.

다양한 시장 상황에서 검증한다.

* 상승장
* 하락장
* 횡보장

---

# 19. 개발 원칙

백테스트는 개발 완료 후 수행하는 작업이 아니다.

설계 단계부터 백테스트를 고려하여 Rule을 작성한다.

# 핵심 원칙

모든 Rule은 백테스트로 증명되어야 한다.

증명되지 않은 Rule은 TASS에 포함하지 않는다.

TASS는 '좋아 보이는 시스템'이 아니라 '검증된 시스템'을 목표로 한다.
